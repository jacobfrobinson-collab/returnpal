Bulk spreadsheet import

Collapsed by default — expand when uploading. Use Preview before saving.

Marketplace / eBay — column help

Upload a .csv or .xlsx where each row is one sale. The Client ID column must match the client’s dashboard ID (e.g. 0014) or Old Client ID (e.g. AC).

Unknown Client IDs can go to Pending import rows when the queue option is on.

Columns: Client ID, sold_date, item_name, quantity, earnings. Optional: order_number.

Returns / refunds: Each row must match an existing sale on that client’s dashboard (same order_number + product title, or set linked_sold_item_id). Cancelled orders that were never imported as sales are skipped so they do not reduce payout.

eBay refunds file → client dashboards & payouts

Upload your eBay Refunds report plus your EVERY EBAY ORDER SHEET. Client IDs are filled automatically; review and edit any row before importing. Payouts use each row’s refund date.

Where do rows go?
  • Ready to import — Client ID matches a ReturnPal account → refund goes on their dashboard when you click Import reviewed rows.
  • Unknown client (blue message) — Order is remembered; refund goes to Pending import rows until that Old Client ID exists, then you apply them.
  • Already imported — Same order/amount/date as before → skipped (filter Already imported).
  • While Show → Needs Client ID is on, rows disappear from the list when you fix them — switch to Ready to import or All rows to see them.
